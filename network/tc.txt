Here are the important tc commands explained in very simple English with examples.
tc = Traffic Control in Linux.
Used for bandwidth limit, delay, packet loss, queue control.
Very important for networking labs.

1. Add network delay

Example: add 100ms delay on interface eth0

sudo tc qdisc add dev eth0 root netem delay 100ms

2. Add delay with variation (jitter)
sudo tc qdisc add dev eth0 root netem delay 100ms 20ms


100ms delay + ±20ms variation.

3. Add packet loss
sudo tc qdisc add dev eth0 root netem loss 10%


Drops 10% packets randomly.

4. Add packet duplication
sudo tc qdisc add dev eth0 root netem duplicate 5%


Duplicated packets → 5%.

5. Add corruption (bit errors)
sudo tc qdisc add dev eth0 root netem corrupt 1%

6. Add bandwidth limit

Use tbf queue discipline.

Example: limit speed to 1 Mbps

sudo tc qdisc add dev eth0 root tbf rate 1mbit burst 32kbit latency 400ms

7. Add multiple rules (delay + loss)
sudo tc qdisc add dev eth0 root netem delay 50ms loss 5%

8. Show current tc rules
tc qdisc show dev eth0

9. Delete all rules
sudo tc qdisc del dev eth0 root

10. Use tc with ingress (incoming traffic)

Add ingress qdisc:

sudo tc qdisc add dev eth0 ingress

11. Limit download speed
sudo tc filter add dev eth0 parent ffff: protocol ip prio 50 u32 \
match ip dst 0.0.0.0/0 police rate 2mbit burst 100k drop flowid :1


This limits incoming traffic to 2 Mbps.
